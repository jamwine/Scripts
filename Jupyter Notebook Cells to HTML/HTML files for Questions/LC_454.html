454. 4Sum II

Given four lists A, B, C, D of integer values, compute how many tuples (i, j, k, l) there are such that A[i] + B[j] + C[k] + D[l] is zero.

To make problem a bit easier, all A, B, C, D have same length of N where 0 ≤ N ≤ 500. 

Example 1:

Input:
A = [ 1, 2]
B = [-2,-1]
C = [-1, 2]
D = [ 0, 2]

Output:
2

Explanation:
The two tuples are:
1. (0, 0, 0, 1) -> A[0] + B[0] + C[0] + D[1] = 1 + (-2) + (-1) + 2 = 0
2. (1, 1, 0, 0) -> A[1] + B[1] + C[0] + D[0] = 2 + (-1) + (-1) + 0 = 0

Solution

class Solution:
    def fourSumCount(self, A: List[int], B: List[int], C: List[int], D: List[int]) -> int:
        result = 0
        hashmap = {}
        for a in A:
            for b in B:
                hashmap[a + b] = hashmap.get(a + b, 0) + 1
        for c in C:
            for d in D:
                result += hashmap.get(-(c + d), 0)
        return result

Time complexity : O(n^2). We have 2 nested loops to count sums, and another 2 nested loops to find complements.

Space complexity : O(n^2) for the hashmap. There could be up to O(n^2) distinct a + b keys.
